# 実装計画

- [x] 1. シークバー性能の改善 — データ構造とプレーヤーの最適化
- [x] 1.1 録画データの最大ユニバース数を動的に検出する
  - ファイル読み込み処理内で全パケットを走査し、出現する最大ユニバース番号+1を記録する
  - パケットリストの型をインデックスアクセス可能な読み取り専用リストに変更し、二分探索を可能にする
  - ユニバース数が0の場合は最低値1を保証するバリデーションを追加する
  - 既存のインターフェースとの後方互換性を維持する
  - _Requirements: 1.1, 1.4_

- [x] 1.2 DMXデータ探索を二分探索に最適化する
  - 再生ヘッダーに対応するパケットを二分探索で検出する処理に置き換える
  - 連続再生時に前回のインデックスから前方探索するキャッシュ最適化を併用する
  - ユニバース番号がバッファサイズを超過する場合のスキップ処理を追加する
  - バッファ初期化を動的ユニバース数に基づいて行うよう変更する
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 1.3 アプリケーション層のバッファ初期化を動的ユニバース数対応にする
  - ハードコードされた固定ユニバース数を、録画データから取得した実ユニバース数に置換する
  - ビジュアライザーのComputeShaderディスパッチに必要な32の倍数への切り上げ処理を確認・調整する
  - タスク1.1および1.2の成果物に依存するため、それらの完了後に実施する
  - _Requirements: 1.1, 1.4_

- [ ] 2. ループ再生機能の実装
- [x] 2.1 ループ再生トグルUIを追加する
  - プレーヤー画面にループ再生のON/OFFを切り替えるトグルボタンを追加する
  - トグルの変更イベントをリアクティブストリームとして公開する
  - _Requirements: 2.1_

- [x] 2.2 ループ再生ロジックをアプリケーション層に実装する
  - ループ有効フラグを管理し、トグルUIからのイベントで切り替える
  - 再生位置が録画データの終端に到達した場合、ループ有効時は先頭にリセットして再生を継続する
  - ループ無効時は再生を停止し、シークバーを終端位置に保持する
  - 再生中にループ設定が変更された場合、即座に新しい設定を反映する
  - タスク2.1のUI公開イベントに依存する
  - _Requirements: 2.2, 2.3, 2.4_

- [ ] 3. ArtNetタイムコード受信による再生制御の実装
- [x] 3.1 (P) ArtNet OpCode定義にタイムコードを追加する
  - OpCode列挙型にTimecodeエントリ (0x97) を追加する
  - 既存のOpCode判定処理との互換性を確認する
  - _Requirements: 3.2_

- [x] 3.2 タイムコード受信コンポーネントを新規作成する
  - バックグラウンドスレッドでポート6454のUDPパケットを受信するループを実装する
  - OpTimeCodeパケット (19バイト) を検出・パースし、タイムコードフィールドを抽出する
  - タイムコード値をミリ秒に変換する処理を実装する（フレームレート種別に応じた変換式を使用）
  - スレッド間通信用のキューを介してメインスレッドにタイムコードデータを受け渡す
  - パケット長不足やOpCode不一致の場合はスキップする
  - ソケットの開始・停止を明示的に管理し、ポート競合時のエラーハンドリングを行う
  - タスク3.1のOpCode定義追加に依存する
  - _Requirements: 3.2, 3.5_

- [x] 3.3 タイムコード受信タイムアウト検出を実装する
  - 最終受信からの経過時間を監視し、閾値（デフォルト2秒）を超えた場合にタイムアウト状態を公開する
  - タイムアウト閾値をInspectorから調整可能にする
  - タスク3.2のコンポーネント内に組み込む
  - _Requirements: 3.5_

- [x] 3.4 タイムコード受信モードUIを追加する
  - プレーヤー画面にタイムコード受信モードのON/OFFトグルを追加する
  - 受信中のタイムコード値をリアルタイム表示するテキスト要素を追加する
  - トグル変更イベントをリアクティブストリームとして公開する
  - タイムコード表示の表示/非表示および内容更新のメソッドを提供する
  - 手動再生コントロール（再生ボタン・スライダー）の有効/無効切替メソッドを提供する
  - _Requirements: 3.1, 3.3, 3.4, 3.6_

- [x] 3.5 タイムコード同期再生ロジックをアプリケーション層に統合する
  - タイムコード受信モードの有効/無効フラグを管理し、UIトグルからのイベントで切り替える
  - モード有効時は、毎フレームのUpdate内でタイムコードキューからデータを取得し、対応する再生位置にシークしてDMXデータを送信する
  - モード有効時は手動再生コントロールを無効化する
  - モード無効時は手動再生コントロールを復元し、現在の再生位置を保持する
  - タイムアウト検出時は最後のタイムコード位置で再生を一時停止する
  - 録画データがロード済みであることを検証してからモードを有効化する
  - タスク3.2, 3.3, 3.4の成果物に依存する
  - _Requirements: 3.2, 3.3, 3.5, 3.6_

- [x] 4. (P) ArtNet再送信先設定の永続化を実装する
  - DstIPフィールドの入力値をローカルストレージに保存し、起動時に読み込む処理を追加する
  - DstPortフィールドの入力値をローカルストレージに保存し、起動時に読み込む処理を追加する
  - 再送信トグルのON/OFF状態をローカルストレージに保存し、起動時に復元する処理を追加する
  - 保存データが存在しない場合のデフォルト値（DstIP: "2.0.0.1"、DstPort: 6454、Toggle: OFF）を設定する
  - IPアドレスとポート番号の値は検証成功時のみ保存し、起動時の読み込みでも検証を行い不正値はデフォルトにフォールバックする
  - 値の変更時に即時保存を行う
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_
